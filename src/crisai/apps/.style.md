Build the interface in a UCL-inspired style, following UCL’s public brand and design system.

Overall feel:
- Clean, institutional, confident, modern.
- Prioritise accessibility, clarity and strong hierarchy.
- Use generous spacing, simple layouts, and strong typographic contrast.
- Avoid overly rounded, playful, glassmorphism, gradients-heavy or SaaS-dashboard styling.

Colour palette:
- Primary dark purple: #361a54
- Off-white background: #fafafa
- Bright purple accent: #993bff
- Mid purple: #ba82ff
- Light purple background: #ddbdff
- Pale purple background: #eedeff
- Heritage blue accent: #30d6ff

Colour rules:
- Lead with dark purple, off-white and purple accents.
- Use bright purple mainly with white/off-white.
- Use mid purple on dark purple where contrast matters.
- Use light/pale purple for panels, callouts and background sections.
- Use heritage blue sparingly for CTAs, highlights or active states.
- Ensure WCAG-compliant contrast.

Typography:
- Use UCL Sans if available.
- Fallback to Aptos, then system sans-serif.
- Headings: semi-bold or medium.
- Body: regular/light, highly readable.
- Headline line-height: 1.1.
- Subheading line-height: 1.2.
- Body line-height: 1.4.
- Use slight negative letter-spacing only on large headings, e.g. -0.04em.

Layout:
- Use a strong header/navigation bar with UCL-style institutional clarity.
- Prefer structured sections: hero, content cards, news/list panels, calls to action.
- Use rectangular or lightly softened cards; avoid very rounded corners.
- Use image-led cards where relevant.
- Keep content blocks modular and easy to scan.
- Use clear breadcrumbs or section navigation where appropriate.

Components:
- Buttons should be bold, simple and high contrast.
- Primary button: dark purple background with off-white text.
- Secondary button: outlined dark purple or light purple background.
- Links should be visually clear and underlined or strongly contrasted.
- Cards should use off-white or pale purple backgrounds, with clear headings and concise copy.
- Use strong focus states for keyboard navigation.

Brand motifs:
- Subtly reference UCL’s “graphic pillars” idea using vertical blocks, stepped bars or column motifs.
- Use these as decorative accents, dividers or hero overlays, not as clutter.
- Do not misuse or recreate the UCL logo unless official assets are provided.

Accessibility:
- Include skip-to-content.
- Use semantic HTML.
- Maintain keyboard navigability.
- Use visible focus rings.
- Do not rely on colour alone.
- Respect reduced-motion preferences.
- Avoid flickering or aggressive animation.

Tone:
- Interface copy should be clear, direct and purposeful.
- Prefer “Find out more”, “Apply now”, “Explore”, “View details”, “Get started”.
- Keep labels short and action-oriented.

Implementation preference:
- Define design tokens for colour, spacing, typography and radius.
- Build reusable components: Header, Hero, Card, CTA, Section, Footer, Breadcrumb.
- Keep the design system consistent rather than styling each page individually.
