## Design overview
- Name: Integration Patterns
- Description: Catalogue page for application and system integration patterns at UCL.
- Classification: integration patterns catalogue; consumer, producer, and ingestion groups.

## When to use
- Use this catalogue to find the integration pattern group and numbered pattern that matches the integration scenario.
- Use it as the starting point for drilling down to leaf pattern pages.

## Implementation
- Consumer patterns listed: Pattern 0 Direct; Pattern 1 Enterprise Channel to System: Synchronous; Pattern 2 Enterprise Channel to System: Asynchronous; Pattern 3 Enterprise API to System: Synchronous; Pattern 4 Enterprise API to System: Asynchronous.
- Producer patterns listed: Pattern 1 System to Enterprise API: onDemand Synchronous; Pattern 2 System to Enterprise Channel: Batch Synchronous; Pattern 3 System to Enterprise Channel: onChange Synchronous.
- Ingestion patterns listed as draft: Pattern 1 Database to Ingestion API; Pattern 2 API to Ingestion API; Pattern 3 DMS (rename) to Ingestion API.
