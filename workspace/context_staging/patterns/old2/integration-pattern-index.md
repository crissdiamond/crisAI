## Design overview
- Catalogue basis: Patterns are reusable building blocks.
- Catalogue basis: They are grouped into areas of use case, such as integration, experience, infrastructure, data and so on.
- Catalogue basis: A pattern can be used within a solution design or to compare a current solution component with a possible new component.
- Catalogue basis: Patterns are defined using names, purpose, key terms, key features, typical use cases, and a diagram showing functional blocks and interactions.
