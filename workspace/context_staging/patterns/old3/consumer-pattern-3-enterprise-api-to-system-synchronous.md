## Design overview
- Name: Enterprise API to System : Synchronous.
- Description: send events received on the Enterprise API to the target system synchronously.
- Classification: consumer pattern.

## When to use
- Use when events are received on the Enterprise API and the target system is to be called synchronously.

## Implementation
- Input: events received on the Enterprise API.
- Action: send to the target system.
- Delivery mode: synchronous.
