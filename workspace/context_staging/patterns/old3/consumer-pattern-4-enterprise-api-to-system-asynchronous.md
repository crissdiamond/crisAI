## Design overview
- Name: Enterprise API to System : Asynchronous.
- Description: send events received on the Enterprise API to the target system asynchronously.
- Classification: consumer pattern.

## When to use
- Use when events are received on the Enterprise API and the target system is to be called asynchronously.

## Implementation
- Input: events received on the Enterprise API.
- Action: send to the target system.
- Delivery mode: asynchronous.
