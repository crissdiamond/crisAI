# UCL integration patterns supporting intranet pages

## Source

- Page title: Integration Patterns
- Web URL: https://liveuclac.sharepoint.com/sites/it-architecture/SitePages/Integration-Patterns(1).aspx
- Open URL: https://liveuclac.sharepoint.com/sites/it-architecture/SitePages/Integration-Patterns(1).aspx

## When to use

- Use this page when you need the high-level statement of what the UCL integration architecture is for.
- Use it to confirm that the integration architecture is technology agnostic and supports exchange between producers and consumers.

## Anti-patterns or when not to use

- Do not use this page for the definitive list of pattern names; the page summary fetched here does not enumerate them.

## Summary from the fetched page

- The integration architecture at UCL facilitates the exchange of data and functionalities between producers and consumers.
- The architecture allows systems and applications to meaningfully interact with each other.
- The architecture is technology agnostic.
- This supports exchange between systems and applications built using heterogeneous technologies.
